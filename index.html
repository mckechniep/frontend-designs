Product Boundary

Finish the vault first. Keep any future app work separate.

The homepage should make the product boundary obvious. Static browsing and brief generation belong here. Auth, jobs, and saved workspaces belong elsewhere later.

Now 01

Public site

Ship the homepage, theme vault, component vault, and brief builder as the finished Framekit experience.

Next 02

Sharper builder

Improve coverage validation, export/shareable briefs, and clearer starter-vs-runtime recommendations.

Later 03

Separate app

If you build generation, previews, or saved workspaces, put them on a separate app surface instead of bloating this repo.

Deployment posture

Deploy the vault like a vault.

Prefer a simple static host for preview branches and production. Keep GitHub Pages only as a fallback if you handle subpaths correctly, and keep any future generator on its own app surface.
