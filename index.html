Product Shape

Framekit is a static vault with a builder, not a hosted generator.

The useful thing here is simple: browse the visual systems, inspect the components, then leave with a brief you can build from. The repo also contains starter and anchor source files, but those are support material for the product, not the product itself.

Finish Line

Finish the static product first. Keep future app work separate.

The repo is strongest when it behaves like a clear public site with honest coverage. Anything involving auth, jobs, saved workspaces, or generation belongs in a separate application later.

Separate app

If you build generation, previews, or saved workspaces, put them on a separate app surface instead of bloating this repo.

Deployment posture

Ship Framekit as a static public site.

Prefer Cloudflare Pages for git previews and simple static deploys. Keep GitHub Pages as the fallback only if you also handle subpath routing correctly. Keep any future generator on a separate app subdomain or repo.
